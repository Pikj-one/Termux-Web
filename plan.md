# Termux Web 交互界面实施计划（plan.md）

## 1. 项目目标

为 Termux 构建一个可在手机浏览器访问的 Web 交互界面，实现：
- 发送命令到 Termux 执行
- 实时查看命令输出（标准输出/错误输出）
- 管理常用命令、历史命令
- 便捷访问 `storage/downloads`
- 在 **无 Root** 环境下稳定运行

---

## 2. 约束与原则

### 2.1 运行环境约束
- 平台：Android + Termux
- 权限：无 Root
- 可用目录重点：`storage/downloads`
- 需兼容手机本地网络与浏览器访问

### 2.2 安全约束（首期必须）
- 服务端仅监听 `127.0.0.1`（默认）
- 不开放局域网，仅允许本机浏览器访问（无需 Token/口令认证）
- 禁止在前端明文持久化高敏感信息
- 命令执行策略：直接执行任意 shell 命令（由本机用户自行承担操作风险）

### 2.3 设计原则
- 最小可用优先（MVP）
- 先稳定后扩展
- 不做兼容性兜底逻辑（按当前目标环境直接实现）

---

## 3. 功能范围规划

## 3.1 MVP（第一阶段）
1. Web 页面基础框架（命令输入、输出窗口、状态栏）
2. 单命令执行接口（提交 -> 执行 -> 返回）
3. 流式输出（WebSocket/SSE 二选一）
4. 命令历史（内存 + 本地文件落盘）
5. 工作目录显示与切换（限制在允许目录内）
6. 文件浏览基础能力（至少支持 `storage/downloads`）

## 3.2 第二阶段（已完成）
1. 多会话/多终端标签页
2. 常用命令收藏与分组
3. 文件上传/下载（与 `storage/downloads` 联动）
4. 命令中断（SIGINT）
5. 执行状态可视化（运行中、成功、失败、耗时）

## 3.3 第三阶段（当前阶段）
1. 任务队列与后台任务
2. 简易权限模型（访客/管理员）
3. 日志检索与导出
4. 移动端 UI 主题与快捷操作优化

---

## 4. 技术架构方案

## 4.1 总体架构
- 前端：移动优先 Web UI（轻量框架或原生）
- 后端：本地 HTTP 服务（Termux 内运行）
- 执行层：子进程执行 shell 命令并捕获 stdout/stderr
- 通信层：
  - 控制面：REST API（提交命令、查询历史）
  - 数据面：WebSocket（实时输出）

## 4.2 技术栈
- 后端：Node.js（Express + ws）
- 前端：Vite + Vue
- 存储：JSON 文件（历史/配置）
- 进程执行：`child_process.spawn`

---

## 5. 模块拆分

1. **web-server 模块**
   - 启动 HTTP 服务
   - 路由注册、静态资源托管

2. **command-runner 模块**
   - 命令执行/终止
   - 输出分流（stdout/stderr）
   - 会话生命周期管理

3. **realtime-gateway 模块**
   - WebSocket 连接管理
   - 输出广播与订阅

4. **history-store 模块**
   - 命令历史写入/读取/清理
   - 最近使用统计

5. **file-browser 模块**
   - 目录列表、路径校验
   - 限定访问根目录（含 `storage/downloads`）

---

## 6. API 草案（首版）

### 6.1 命令执行
- `POST /api/exec`
  - 入参：`{ command: string, cwd?: string }`
  - 返回：`{ sessionId: string }`

### 6.2 会话状态
- `GET /api/sessions/:id`
  - 返回运行状态、开始时间、退出码

### 6.3 终止执行
- `POST /api/sessions/:id/kill`
  - 行为：向进程发送中断信号

### 6.4 历史记录
- `GET /api/history`
- `POST /api/history/pin`

### 6.5 文件浏览
- `GET /api/files?path=...`
  - 必须进行路径规范化和越界检查

### 6.6 实时输出
- `WS /ws/sessions/:id`
  - 消息：`stdout`、`stderr`、`exit`

## 8. 关键实现细节

1. **命令执行安全**
   - 默认不拼接用户输入到高危模板
   - 对危险命令（如 rm、dd 等）做提示确认

2. **路径安全**
   - 所有路径通过 `resolve + normalize` 后校验
   - 禁止 `..` 越权访问不允许目录

3. **会话与输出**
   - 每次执行创建 sessionId
   - 输出按时间顺序推送并落盘（可选）

4. **移动端体验**
   - 输入框固定底部
   - 输出区虚拟滚动/分页（防卡顿）

5. **稳定性**
   - 限制并发会话数
   - 大输出时背压与截断策略

## 10. 测试计划

1. **功能测试**
   - 执行成功命令、失败命令、长时间命令
   - 中断命令后状态正确
   - 历史记录读写一致

2. **边界测试**
   - 超长输出、超长命令
   - 非法路径、越权路径
   - WebSocket 断线重连

3. **安全测试**
   - 命令注入尝试
   - 路径穿越尝试
   - 验证仅 `127.0.0.1` 监听，局域网设备不可访问

4. **移动端测试**
   - 小屏布局
   - 输入法遮挡
   - 前后台切换恢复

---

## 11. 风险与应对

1. **大输出导致页面卡顿**
   - 应对：分块渲染、行数上限、分页显示

2. **命令误执行风险**
   - 应对：界面固定风险提示、默认工作目录可见、命令历史可追溯

3. **监听地址配置错误导致暴露风险**
   - 应对：固定仅本地 `127.0.0.1` 监听，不提供局域网暴露配置

4. **Termux 进程资源限制**
   - 应对：并发限制与任务队列

---

## 12. 验收标准（Definition of Done）

满足以下即视为首版完成：
- 可在手机浏览器稳定打开 Web 界面
- 可执行并实时查看命令输出
- 可查看历史命令并复用
- 可访问并浏览 `storage/downloads`
- 基础安全策略（固定本地 127.0.0.1 监听）生效
- 命令执行策略与产品约定一致（直接执行任意 shell 命令）
